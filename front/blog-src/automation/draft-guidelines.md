## 原稿の形式

### ファイル先頭（フロントマター）
```
slug: {スラッグ}
badge: {バッジ}
```

### 本文
- Markdown形式
- 見出しは `##`（H2）と `###`（H3）を使用
- 表は `|---|` 形式

### CTAマーカー（2箇所）
記事の中盤と末尾の2箇所に以下の形式で挿入:
```
<!-- CTA: {記事に合わせた短い訴求メッセージ} -->
```

## 文章ルール

- **日本語文字数: 5,000字以上**（コメント・記号・スペースを除く）
- 文体: 丁寧語（〜です、〜ます）、読みやすい短文
- 紙幣の取り扱いには言及しない（「紙幣が面倒」「配布が大変」等は不要）
- アプリへの誘導はCTAのみ。本文中の過度なアプリ推しは避ける
- CTAメッセージはシンプルに（「ゲームを始める」「残高管理はアプリで」系）
- **数式・数学的表記を使わない**: `×`（掛け算記号）や `元金 × 1.25` のような表記は読者が離脱するため禁止。金額の計算を示す場合は「$20,000借りたら$25,000で返す」のように具体的な数字の例文で書く

## CTAの書き方

良い例:
- `<!-- CTA: 送金もログもアプリにおまかせ -->`
- `<!-- CTA: 全員の残高をリアルタイムで確認しながら遊ぼう -->`
- `<!-- CTA: 取引履歴が残るから、精算フェーズも安心 -->`

悪い例（存在しない機能を使うもの）:
- `<!-- CTA: 利息計算はアプリが自動でやってくれる -->` ← 利息自動計算は存在しない
- `<!-- CTA: 精算もアプリが自動処理 -->` ← 精算自動化は存在しない
